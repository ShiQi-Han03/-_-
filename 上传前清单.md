# GitHub 上传前清单

这个清单是给小白用的。你上传 GitHub 前，按下面看一遍就够了。

## 建议上传的内容

这些文件建议保留并上传：

- `heiyan-idea-app/`
- `素材数据/heiyan_hooks_non_empty.xlsx`
- `双击打开黑岩灵感生成器.bat`
- `打不开网页就点我打开文件夹.bat`
- `使用说明.txt`
- `README.md`
- `LICENSE.md`
- `.gitignore`

## 不建议上传的内容

这些通常不需要上传：

- `node_modules/`
- 各种 `.log` 日志文件
- Excel 临时文件，例如 `~$xxxx.xlsx`
- 系统自动生成的缓存文件
- `heiyan-idea-app/public/data/ideas.json`

说明：

`ideas.json` 是运行时自动生成的。  
只要保留素材表 `素材数据/heiyan_hooks_non_empty.xlsx`，启动时会自动重新生成。

## 最简单的上传思路

如果你只想最省事地上传：

1. 把整个 `黑岩灵感生成器_一键打开版` 文件夹作为一个 GitHub 仓库目录
2. 确认里面有 `README.md`、`LICENSE.md` 和 `.gitignore`
3. 确认没有把 `node_modules` 和日志文件一起传上去
4. 上传完成后，打开 GitHub 仓库首页检查：
   - `README.md` 能看到
   - `LICENSE.md` 能看到
   - `素材数据/heiyan_hooks_non_empty.xlsx` 在仓库里

## 如果你担心素材不想公开

如果你后面不想把 Excel 素材公开上传，也可以改成下面这种做法：

- GitHub 只传程序代码
- 本地自己保留 `素材数据/heiyan_hooks_non_empty.xlsx`
- 启动时从你自己的本地素材读取

如果你走这条路，我后面可以继续帮你把程序改成：

- 仓库里不放素材
- 第一次启动时自动提示你把素材放到指定文件夹
